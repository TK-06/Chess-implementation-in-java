How Each Pattern Is Implemented

Observer Pattern

  • Board is the subject because it stores and notifies a list of GameObserver objects.
  • MoveLogger, CheckDetector, and ChessGUI all implement GameObserver.
  • This keeps logging, check detection, and GUI updates separate from the board state itself.

Factory Pattern

  • PieceFactory.create(...) is the single construction point for all chess pieces.
  • GameManager.initBoard() and GameManager.promote(...) both use the factory.
  • This avoids spreading piece constructor logic throughout the project.

Singleton Pattern

  • GameManager has a private constructor and a static instance field.
  • getInstance() returns one shared game controller for the whole app.
  • Main, ChessGUI, and BoardPanel all use the same shared object.
